## Компоненты и пропсы

### Как создавать функциональные и классовые компоненты в React?

В React есть два основных способа создания компонентов: функциональные компоненты и классовые компоненты. 

**Функциональные компоненты** представляют собой простые JavaScript-функции, которые принимают пропсы (свойства) в качестве аргумента и возвращают JSX-код, описывающий внешний вид компонента.

**Пример функционального компонента:**

```jsx
function Welcome(props) {
  return <h1>Привет, {props.name}!</h1>;
}
```

**Классовые компоненты** более сложные и наследуются от класса `React.Component`. Они имеют доступ к состоянию (state) и жизненному циклу компонента.

**Пример классового компонента:**

```jsx
class Welcome extends React.Component {
  render() {
    return <h1>Привет, {this.props.name}!</h1>;
  }
}
```

**Начиная с версии 16.8 React, функциональные компоненты с использованием хуков (hooks) могут полностью заменить классовые компоненты.**

### Как передавать пропсы (свойства) в компоненты, и почему это важно для работы с данными в React?

Пропсы – это механизм передачи данных от родительских компонентов к дочерним.  Они передаются как атрибуты HTML-тегов при использовании компонента.

**Пример передачи пропсов:**

```jsx
function App() {
  return (
    <div>
      <Welcome name="Иван" /> 
      <Welcome name="Мария" />
    </div>
  );
}
```

В данном примере компонент `App` передает пропс `name` компонентам `Welcome`.

**Важность пропсов:**

* **Однонаправленный поток данных:** Пропсы обеспечивают предсказуемый поток данных от родительских компонентов к дочерним, что упрощает понимание и отладку кода.
* **Повторное использование компонентов:**  Благодаря пропсам, один и тот же компонент может использоваться с разными данными, что делает код более гибким и удобным в поддержке.

### Каким образом можно использовать деструктуризацию пропсов для улучшения читаемости кода?

Деструктуризация пропсов позволяет извлекать значения пропсов в отдельные переменные, что делает код более читаемым и лаконичным.

**Пример деструктуризации пропсов:**

```jsx
function Welcome({ name, age }) { // Деструктуризация пропсов name и age
  return (
    <div>
      <h1>Привет, меня зовут {name}</h1>
      <p>Мне {age} лет</p>
    </div>
  );
}
```

В данном примере мы деструктурируем пропсы `name` и `age` прямо в аргументах функции, что делает код более компактным и понятным.

## Сужения типов

### В чем заключается концепция "сужения типов" в контексте React и TypeScript?

**Сужение типов (Type Narrowing)** в контексте React и TypeScript — это процесс уточнения типа переменной от более общего к более конкретному. Это важно для того, чтобы TypeScript мог проводить более точную проверку типов и предотвращать ошибки, связанные с некорректным использованием данных.

В React часто приходится работать с данными, тип которых может быть неизвестен заранее, например, данные, полученные от API. В таких случаях TypeScript присваивает этим данным тип `any`, что отключает проверку типов. Сужение типов позволяет нам указать TypeScript, какой именно тип данных мы ожидаем, и тем самым включить проверку типов.

**Примеры:**

**1. Проверка на `null` или `undefined`:**

```typescript
interface User {
  name: string;
  age: number;
}

function greetUser(user: User | null) {
  // TypeScript не знает, является ли user null или нет
  // user.name.toUpperCase() // Ошибка! user может быть null

  if (user) {
    // Внутри блока if TypeScript знает, что user не null
    console.log(user.name.toUpperCase()); // OK
  }
}
```

**2. Использование оператора `in`:**

```typescript
interface Dog {
  bark(): void;
}

interface Cat {
  meow(): void;
}

function makeSound(animal: Dog | Cat) {
  // TypeScript не знает, является ли animal собакой или кошкой
  // animal.bark(); // Ошибка! animal может быть кошкой

  if ('bark' in animal) {
    // Внутри блока if TypeScript знает, что animal - это Dog
    animal.bark(); // OK
  } else {
    // В противном случае animal - это Cat
    animal.meow(); // OK
  }
}
```

**3. Пользовательские защитники типов (Type Guards):**

```typescript
interface Product {
  id: number;
  name: string;
  price: number;
}

interface CartItem {
  product: Product;
  quantity: number;
}

function isProduct(item: Product | CartItem): item is Product {
  return (item as Product).price !== undefined;
}

function calculateTotalPrice(items: (Product | CartItem)[]) {
  let totalPrice = 0;
  for (const item of items) {
    if (isProduct(item)) {
      totalPrice += item.price; // OK
    } else {
      totalPrice += item.product.price * item.quantity; // OK
    }
  }
  return totalPrice;
}
```

В данном примере функция `isProduct` является пользовательским защитником типов. Она проверяет, имеет ли объект свойство `price`. Если да, то функция возвращает `true` и TypeScript знает, что объект имеет тип `Product`.

Использование сужения типов делает код React и TypeScript более безопасным, понятным и удобным в поддержке. 

### Как использовать условные типы для более точного определения типов данных внутри компонентов React?

Условные типы (Conditional Types) позволяют создавать типы данных, которые зависят от выполнения определенных условий. 

**Пример условного типа:**

```typescript
interface User {
  name: string;
  age?: number; // Необязательное свойство
}

type UserWithAge = User extends { age: number } ? User : never;

const user1: User = { name: "Иван" };
const user2: UserWithAge = { name: "Мария", age: 30 }; // OK
const user3: UserWithAge = { name: "Петр" }; // Ошибка: отсутствует свойство "age"
```

В данном примере мы создали тип `UserWithAge`, который будет соответствовать типу `User` только в том случае, если объект содержит свойство `age`.

### Как избежать ошибок сужения типов и обеспечить безопасность типов в процессе разработки React-приложений?

* **Использовать TypeScript:**  TypeScript позволяет выявлять ошибки типов на этапе компиляции, что значительно повышает надежность кода.
* **Явно указывать типы данных:**  Чем больше типов данных вы укажете явно, тем меньше вероятность ошибок, связанных с неявным приведением типов.
* **Использовать линтеры и форматеры кода:**  Инструменты, такие как ESLint и Prettier, помогают поддерживать единообразие кода и выявлять потенциальные проблемы, связанные с типизацией.
* **Писать тесты:**  Unit-тесты позволяют проверить, что код работает корректно с разными типами данных и предотвратить регрессию ошибок в будущем.

## Типы на каждый день

### Как создавать пользовательские типы данных для обеспечения ясности и понятности кода в повседневной разработке React-приложений?

Пользовательские типы данных (Custom Types) позволяют создавать собственные именованные типы, что делает код более читаемым и понятным. 

**Пример пользовательского типа:**

```typescript
interface Product {
  id: number;
  name: string;
  price: number;
}
```

В данном примере мы создали интерфейс `Product`, который описывает структуру данных для продукта. 

### Как использовать enum и type union для удобного представления различных состояний в приложении?

**Enum (перечисления)** позволяют создавать типы данных, которые могут принимать значения только из определенного списка.

**Пример использования enum:**

```typescript
enum Status {
  Pending = "pending",
  Success = "success",
  Error = "error",
}

interface Request {
  status: Status;
}
```

**Type Union (объединение типов)** позволяет комбинировать несколько типов данных в один.

**Пример использования type union:**

```typescript
type Result = string | number;

const result1: Result = "Успех"; // OK
const result2: Result = 100; // OK
const result3: Result = {}; // Ошибка: тип не соответствует Result
```

**Использование enum и type union делает код более выразительным и помогает избежать ошибок, связанных с использованием некорректных значений.**
