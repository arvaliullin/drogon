### Аннотация Типа

#### 1. Какие преимущества предоставляет TypeScript в сравнении с JavaScript при использовании аннотаций типов?

**TypeScript** предоставляет следующие преимущества:
- **Раннее выявление ошибок**: Аннотации типов позволяют выявлять ошибки в коде на этапе компиляции, что значительно снижает количество ошибок в рантайме.
- **Улучшенная читаемость и документация**: Благодаря аннотациям типов код становится более понятным и самодокументируемым.
- **Поддержка инструментов**: IDE могут лучше поддерживать рефакторинг и подсказки благодаря статическому анализу типов.

**Пример кода:**

```typescript
function add(a: number, b: number): number {
    return a + b;
}

// Ошибка на этапе компиляции
add(10, "20"); // Argument of type 'string' is not assignable to parameter of type 'number'.
```

#### 2. Какие базовые типы данных доступны в TypeScript, и как они могут быть использованы для определения переменных?

**Базовые типы данных в TypeScript**:
- `number`
- `string`
- `boolean`
- `array`
- `tuple`
- `enum`
- `any`
- `void`
- `null` и `undefined`
- `never`

**Пример использования:**

```typescript
let count: number = 10;
let name: string = "Alice";
let isActive: boolean = true;
let numbers: number[] = [1, 2, 3];
let tupleExample: [string, number] = ["Bob", 25];
```

#### 3. Почему использование аннотаций типов может помочь в предотвращении ошибок во время компиляции?

Аннотации типов позволяют:
- **Выявлять несоответствия типов**: Ошибки типа сразу обнаруживаются компилятором.
- **Предупреждать о возможных ошибках логики**: Логические ошибки, такие как передача неправильных аргументов, могут быть пойманы раньше.

**Пример:**

```typescript
let value: string = "Hello";

// Ошибка на этапе компиляции
value = 5; // Type 'number' is not assignable to type 'string'.
```

### Псевдонимы типов, объединения и пересечения

#### 1. Как создать и использовать псевдонимы типов в TypeScript?

**Псевдонимы типов** создаются с помощью ключевого слова `type` и могут быть использованы для упрощения сложных типов или добавления читаемости.

**Пример кода:**

```typescript
type StringOrNumber = string | number;

let value1: StringOrNumber = "Hello";
let value2: StringOrNumber = 42;
```

#### 2. Как объединять и пересекать типы данных, и в каких сценариях это может быть полезно?

**Объединение типов** (`Union types`) позволяет переменной принимать значения из нескольких типов.

**Пересечение типов** (`Intersection types`) позволяет комбинировать несколько типов в один.

**Пример кода:**

```typescript
// Объединение типов
type ID = string | number;

// Пересечение типов
interface Person {
    name: string;
}

interface Employee {
    employeeId: number;
}

type EmployeePerson = Person & Employee;

let emp: EmployeePerson = {
    name: "Alice",
    employeeId: 123
};
```

**Сценарии использования:**
- Объединение типов используется, когда переменная может принимать значения разных типов (например, ID может быть числом или строкой).
- Пересечение типов полезно для комбинирования свойств нескольких типов в одном (например, EmployeePerson объединяет свойства Person и Employee).

#### 3. В чем различие между интерфейсами и псевдонимами типов, и когда лучше использовать одно из них?

**Интерфейсы** и **псевдонимы типов** имеют сходные возможности, но различаются в использовании:
- **Интерфейсы** лучше подходят для описания объектов и могут быть расширены (через `extends`).
- **Псевдонимы типов** более универсальны и могут описывать объединения, пересечения и простые типы.

**Пример кода:**

```typescript
interface Animal {
    name: string;
}

interface Dog extends Animal {
    breed: string;
}

type Pet = {
    name: string;
    age: number;
};
```

**Когда использовать:**
- Используйте **интерфейсы** для описания объектов и классов.
- Используйте **псевдонимы типов** для объединения, пересечения и более сложных типов не-объектной природы.
